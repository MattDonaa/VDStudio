From Block Coding to Python: A Learner's Journey, Grade R to Matric

What should a five-year-old, a ten-year-old and a sixteen-year-old each be learning in coding? A stage-by-stage map of the journey from blocks to real code.

  1. Home
  2. Resources
  3. From Block Coding to Python: A Learner's Journey, Grade R to Matric

Published 24 June 2026 · 7 min read · by the NovaCode Academy team

Parents often ask when a child should 'really' start coding. The honest answer: computational thinking can start in Grade R, but what it looks like changes completely as a learner grows. Getting the progression right matters more than starting early.

Foundation phase: thinking before typing

From Grade R to Grade 3, coding is about sequencing, patterns and cause-and-effect — often without a screen at all. Unplugged activities and simple visual tools like ScratchJr build the mental models everything else rests on.

Intermediate phase: the block years

Grades 4 to 6 are the golden age of block-based coding. Scratch lets learners build genuinely impressive games and animations while internalising loops, conditionals, variables and events — the grammar of every programming language they will ever meet.

Senior phase and FET: real languages, real projects

From Grade 7, learners are ready for the block-to-text transition — and Python is the ideal first text language: readable, forgiving and genuinely used in industry. By FET phase (Grades 10–12), learners should be building portfolio projects: websites, data projects, apps and robotics builds they can show to universities and employers.

A matric learner who has followed this journey doesn't just have a skill — they have proof of it. That portfolio is worth more than any certificate.
